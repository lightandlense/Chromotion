Create src/runtime/ directory and write src/runtime/part_renderer.js as an ES module using Pixi.js v7 via CDN URL (https://cdn.jsdelivr.net/npm/pixi.js@7/dist/pixi.min.js). This is NOT a modification of src/scene/ram_rig_test.html — build fresh for the 1920x1080 motion_data coordinate system.

Canvas setup:
- PIXI.Application width=1920, height=1080, backgroundAlpha=0, antialias=true, resolution=1, autoDensity=false
- Append app.view to document.getElementById('canvas-container')
- CSS scale to fit window: on resize, compute scale = min(window.innerWidth/1920, window.innerHeight/1080), set app.view.style.width/height accordingly

Export two functions: `init(scanId)` and `cleanupSession(currentState)`.

`init(scanId)` implementation:
1. Fetch /data/motion_data.json and /data/parts_config.json in parallel via Promise.all
2. Build lineArtUrls array: 121 URLs for src/animations/ram_lineart/frame_NNNN.png (zero-padded to 4 digits)
3. Pre-load ALL line-art frames via PIXI.Assets.load(lineArtUrls) — this is startup cost, not per-visitor
4. Pre-load 8 scan texture URLs in parallel: /data/scans/${scanId}/textures/${part}.png for each part in motionData.parts_list
5. Build spriteContainer (PIXI.Container, sortableChildren=true)
6. For each part in motionData.parts_list (use parts_list, NOT Object.keys — preserves order):
   - Create PIXI.Sprite from pre-loaded texture
   - sprite.anchor.set(0.5, 0.5)
   - sprite.zIndex = partsConfig.z_order[partName] ?? 0
   - Add to spriteContainer, store in sprites[partName]
7. spriteContainer.sortChildren() — sort once after all added
8. Add spriteContainer to app.stage
9. Build lineArtContainer (PIXI.Container), create PIXI.Sprite for lineArtUrls[0], set width=1920 height=1080, add to lineArtContainer, add lineArtContainer to app.stage AFTER spriteContainer (renders on top)
10. Animation ticker using delta accumulation (Pattern 6 from research):
    - frameAccum += delta
    - framesPerAnimFrame = 60 / motionData.fps  (= 2.5 at 24fps)
    - When frameAccum >= framesPerAnimFrame: frameAccum -= framesPerAnimFrame; advance currentFrame
    - Per frame: for each part, read motionData.parts[partName].frames[currentFrame]; if frame is missing or tracking_quality === 0, skip that part; set sprite.position.set(frame.cx, frame.cy); sprite.rotation = frame.angle
    - Swap lineArtSprite.texture = PIXI.Texture.from(lineArtUrls[currentFrame]) — synchronous cache hit after pre-load
11. Return { sprites, partNames: motionData.parts_list, textureUrls, app, lineArtUrls }

`cleanupSession(state)` implementation:
- app.ticker.stop()
- For each partName in state.partNames:
  - url = /data/scans/${state.scanId}/textures/${partName}.png
  - PIXI.Texture.from(url).destroy(true)  — destroy GPU memory (true = destroy BaseTexture)
  - await PIXI.Assets.unload(url)         — evict from Assets cache (prevents stale texture on next scan)
- Do NOT destroy line-art textures — they are static baked assets, not visitor-specific
- app.ticker.start() — restart ticker so kiosk can accept next scan

Anti-patterns to avoid:
- Do NOT use AnimatedSprite for part sprites — they have fixed textures and animated transforms
- Do NOT call PIXI.Texture.from() inside the ticker without prior PIXI.Assets.load() — causes async loads and blank frames
- Do NOT set canvas to 1344x768 — this is a new 1920x1080 renderer, not a modification of the prototype
- Do NOT mix the pivot system from ram_rig_test.html — use centroid-based absolute positioning (cx/cy directly from motion_data)

**kiosk.html** — browser entry point at src/runtime/kiosk.html:

Structure:
- div#canvas-container (fullscreen, black background)
- div#loading-overlay (shown during startup lineart pre-load and scan pipeline)
- div#scan-controls (webcam video element + "Scan" button, hidden while animation runs)
- Import part_renderer.js as ES module (type="module")
- Import Pixi.js v7 from CDN: https://cdn.jsdelivr.net/npm/pixi.js@7/dist/pixi.min.js

Startup flow:
1. Show loading overlay "Loading..."
2. Call init() with a demo/default scan_id (use 'demo' or the first scan_id found in data/scans/ — kiosk_server.py provides GET /api/status with current scan_id if one exists)
3. On init() complete, hide loading overlay, show animation
4. Webcam: call navigator.mediaDevices.getUserMedia({video: {width:1920, height:1080}}) and display in video element (hidden behind canvas, used for snapshot on scan)

Scan flow (when "Scan" button clicked):
1. Show loading overlay "Scanning..."
2. Capture webcam frame to canvas, convert to JPEG blob (canvas.toBlob with 'image/jpeg', 0.92)
3. POST blob to /api/scan with Content-Type: image/jpeg
4. Response: { status: 'ok', scan_id: 'xxx' } or { status: 'error', message: '...' }
5. On error: show message to visitor ("Please try again — [message from server]"), hide overlay
6. On ok: poll GET /api/scan/${scan_id}/status every 500ms
7. When status is 'ready': call cleanupSession(currentState) then init(scan_id)
8. On init complete: hide overlay, resume animation

Do NOT pre-warn about file:// CORS — kiosk.html is always served via kiosk_server.py (RENDER-06).

**kiosk_server.py** — Python server at ops/kiosk_server.py:

Implement using Python http.server stdlib (no Flask unless needed). Serve the project root as static files. Handle:

POST /api/scan:
- Read body bytes (Content-Length header), write to data/scans/<uuid8>/raw_scan.jpg
- Run subprocess: python src/preprocess/scan_rectify.py --input <raw> --output <rectified>
  - If returncode != 0: send JSON {status: 'error', message: stdout.strip()}, 400
- Run subprocess: python src/preprocess/scan_slice.py --scan <rectified> --masks-dir data/rest_pose_masks --output-dir data/scans/<uuid8>/textures
- Send JSON {status: 'ok', scan_id: '<uuid8>'}, 200
- Also write data/scans/latest_scan_id.txt with scan_id for GET /api/status

GET /api/scan/<scan_id>/status:
- Check if data/scans/<scan_id>/textures/ exists and contains 8 .png files
- If yes: send {status: 'ready', scan_id: scan_id}
- If no: send {status: 'processing', scan_id: scan_id}

GET /api/status:
- Return {current_scan_id: <contents of data/scans/latest_scan_id.txt or null>}

Static file serving: SimpleHTTPRequestHandler serving from PROJECT_ROOT (directory=str(PROJECT_ROOT)). Serve all GET requests not matching /api/* as static files.

CORS: Add Access-Control-Allow-Origin: * header on all responses (required for localhost development).

Run on localhost:8000 by default, configurable via PORT env var.

Add shebang and __main__ guard. Print startup message: "Kiosk server at http://localhost:8000"

Use sys.executable for subprocess Python path (ensures same venv/conda env as the server).
