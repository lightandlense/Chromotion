Create src/tests/test_integ_bad_scan.py. This test verifies that a deliberately bad scan produces graceful fallback behavior from the Python pipeline, matching INTEG-04 requirements:
- Dark colors → preserved as-is in the texture (not treated as line art)
- Strokes outside the lines → included in the texture (scan_slice clips to mask boundary only, does not interpret content)
- Missing/uncolored regions → white fallback (1x1 RGBA or white-filled crop), not an error

Test inputs — create synthetic cases with PIL in tmp_path fixtures:

1. `bad_scan_all_black.png` — 1920x1080 PNG, all pixels (0,0,0) — simulates severe underexposure
2. `bad_scan_overexposed.png` — 1920x1080 PNG, all pixels (255,255,255) — simulates all-white / uncolored sheet  
3. `bad_scan_partial.png` — 1920x1080 PNG, 50% black / 50% white — simulates partial scan

All three are pre-rectified (no ArUco detection needed) — pass directly to scan_slice.py.

`test_bad_scan_no_crash(tmp_path)`:
- For each of the 3 bad scan images:
  - Call scan_slice.py via subprocess.run([...]) with --scan=<bad_image> --masks-dir=data/rest_pose_masks --output-dir=tmp_path/textures_<n>
  - Assert returncode == 0 (no crash)
  - Assert 8 output .png files exist in output dir (one per part)
  - Assert all 8 texture_meta_<part>.json files exist

`test_bad_scan_all_parts_have_texture(tmp_path)`:
- Use bad_scan_all_black.png
- Run scan_slice.py
- For each part's output .png:
  - Open with PIL, read first pixel RGBA
  - Assert the file is a valid PNG (PIL.Image.open does not raise)
  - Assert dimensions are at least 1x1
  - Assert NOT zero bytes (file is non-empty)

`test_uncolored_scan_renders_white(tmp_path)`:
- Use bad_scan_overexposed.png (all-white simulates uncolored sheet)
- Run scan_slice.py
- For at least one part texture: open with PIL, check that the dominant color is near-white (R,G,B all > 200) OR is_fallback=true in texture_meta JSON
- This confirms INTEG-04 behavior: "uncolored regions render white, not as errors"

`test_bad_scan_rejection_not_crash(tmp_path)`:
- Call scan_rectify.py on an all-black image with no ArUco markers
- Assert returncode != 0 (rejection) OR returncode == 0 with rejection message in stdout
- Assert NO unhandled exception in stderr (script should exit cleanly, not traceback)
- This confirms RUNTIME-02/03/04 rejection behavior still holds (smoke test from Phase 2 perspective)

All tests must be independently runnable without the kiosk server (direct subprocess calls to scripts, no HTTP).
