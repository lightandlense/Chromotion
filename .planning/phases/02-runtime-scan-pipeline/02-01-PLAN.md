**Function signature (exact):**
```python
def rectify_scan(
    input_path: Path,
    output_path: Path,
    target_w: int = 1920,
    target_h: int = 1080,
) -> tuple[bool, str | None]:
    """
    Rectify a coloring sheet scan using ArUco corner markers.
    Returns (True, None) on success; saves rectified_scan.png to output_path.
    Returns (False, error_message) on rejection; does NOT write any file.
    """
```

**Implementation steps (in order):**

1. Load image with `cv2.imread(str(input_path))`. If None, return `(False, "could not read image")`.

2. Setup ArUco — use new API only (4.10):
   ```python
   aruco_dict = cv2.aruco.getPredefinedDictionary(cv2.aruco.DICT_4X4_50)
   params = cv2.aruco.DetectorParameters()
   detector = cv2.aruco.ArucoDetector(aruco_dict, params)
   gray = cv2.cvtColor(bgr, cv2.COLOR_BGR2GRAY)
   corners, ids, _ = detector.detectMarkers(gray)
   ```

3. Marker check: if `ids is None or len(ids) < 4`, return `(False, "couldn't read corners, try again")`.

4. Build id_map: `{int(mid): corner_arr[0].mean(axis=0) for corner_arr, mid in zip(corners, ids.flatten())}`. If not all 4 IDs 0-3 present, return same prompt.

5. Skew check (BEFORE homography):
   ```python
   top_w    = np.linalg.norm(id_map[1] - id_map[0])   # TR - TL
   bottom_w = np.linalg.norm(id_map[2] - id_map[3])   # BR - BL
   left_h   = np.linalg.norm(id_map[3] - id_map[0])   # BL - TL
   right_h  = np.linalg.norm(id_map[2] - id_map[1])   # BR - TR
   w_dev = abs(top_w - bottom_w) / max(top_w, bottom_w)
   h_dev = abs(left_h - right_h) / max(left_h, right_h)
   if max(w_dev, h_dev) > 0.20:
       return False, "perspective too extreme, please rescan"
   ```

6. Compute homography and warp:
   ```python
   src_pts = np.float32([id_map[0], id_map[1], id_map[2], id_map[3]])
   dst_pts = np.float32([[0,0],[target_w-1,0],[target_w-1,target_h-1],[0,target_h-1]])
   H, _ = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC)
   if H is None:
       return False, "couldn't compute homography, try again"
   warped = cv2.warpPerspective(bgr, H, (target_w, target_h), borderValue=255)
   ```

7. Histogram check (AFTER warp):
   ```python
   gray_warped = cv2.cvtColor(warped, cv2.COLOR_BGR2GRAY)
   median_lum = float(np.median(gray_warped))
   if median_lum < 30:
       return False, "too dim, try again"
   if median_lum > 230:
       return False, "too overexposed, try again"
   ```

8. Save with `cv2.imwrite(str(output_path), warped)`. Return `(True, None)`.

**`main()` function:** Use argparse with `--input` (required), `--output` (default: `rectified_scan.png`). On failure, print the error message and `sys.exit(1)`.

**Imports:** `cv2`, `numpy as np`, `pathlib.Path`, `sys`, `argparse`. No other dependencies.

**CRITICAL — do NOT use:**
- `cv2.aruco.drawMarker` (removed in 4.10 — use `generateImageMarker` in tests)
- The legacy `cv2.aruco.detectMarkers` free function directly (use ArucoDetector class)
- `cv2.aruco.Dictionary_get` (deprecated — use `getPredefinedDictionary`)

Create `tests/preprocess/test_aruco_rectify.py` covering all TEST-04 and RUNTIME-01 through RUNTIME-04 requirements. All fixtures generated at runtime in pure NumPy/OpenCV — no fixture files committed to disk.

**Constants at top of file:**
```python
MARKER_SIZE = 80
PAD = 20           # quiet zone — ArUco detection requires white border; PAD=20 on 1920x1080 is reliable
TARGET_W = 1920
TARGET_H = 1080
```

**Shared fixture `make_synthetic_scan(fill_value=215, apply_skew=None)`:**
- Create `np.full((TARGET_H, TARGET_W, 3), fill_value, dtype=np.uint8)` canvas
- Add two colored blobs so histogram check passes: e.g., `canvas[300:700, 200:700] = [100, 160, 80]` and `canvas[100:300, 800:1200] = [50, 50, 180]`
- Place ArUco markers at 4 corners using `cv2.aruco.generateImageMarker(aruco_dict, mid, MARKER_SIZE, marker_img, 1)`:
  - ID 0 at top-left: row=PAD, col=PAD
  - ID 1 at top-right: row=PAD, col=TARGET_W-PAD-MARKER_SIZE
  - ID 2 at bottom-right: row=TARGET_H-PAD-MARKER_SIZE, col=TARGET_W-PAD-MARKER_SIZE
  - ID 3 at bottom-left: row=TARGET_H-PAD-MARKER_SIZE, col=PAD
- Record `true_centers[mid]` = `[col + MARKER_SIZE/2, row + MARKER_SIZE/2]` (x,y order)
- If `apply_skew` (a 4-point perspective transform tuple), warp the canvas with `cv2.warpPerspective`
- Return `(canvas_bgr, true_centers)`

**Test functions:**

`test_rectify_produces_output(tmp_path)`:
- Call `make_synthetic_scan()`, save to `tmp_path/scan.png` with `cv2.imwrite`
- Call `rectify_scan(tmp_path/"scan.png", tmp_path/"out.png")`
- Assert `result == (True, None)`
- Load output with `cv2.imread(str(tmp_path/"out.png"))`
- Assert shape is `(1080, 1920, 3)` — confirms RUNTIME-01 target resolution

`test_rectify_2px_tolerance(tmp_path)`:
- This is the TEST-04 core test. Markers are placed at KNOWN positions in a flat (no-skew) scan.
- After rectification, re-detect markers in the output.
- For each marker center detected in the output, compute distance to the true expected position in the 1920x1080 target space.
- True positions in target space: ID0=(PAD+MARKER_SIZE/2, PAD+MARKER_SIZE/2), ID1=(TARGET_W-PAD-MARKER_SIZE/2, PAD+MARKER_SIZE/2), ID2=(TARGET_W-PAD-MARKER_SIZE/2, TARGET_H-PAD-MARKER_SIZE/2), ID3=(PAD+MARKER_SIZE/2, TARGET_H-PAD-MARKER_SIZE/2)
- Assert `max(distances) <= 2.0` — 2px tolerance per TEST-04

`test_reject_too_few_markers(tmp_path)`:
- Create blank canvas (fill=215) with only 2 markers (IDs 0 and 1)
- Save to tmp_path, call `rectify_scan`
- Assert result is `(False, "couldn't read corners, try again")`
- Assert output file does NOT exist

`test_reject_extreme_perspective(tmp_path)`:
- Create a scan where the left edge is 25% shorter than the right edge (>20% skew)
- Apply with `cv2.getPerspectiveTransform` to warp the full-marker canvas
- Call `rectify_scan`, assert result is `(False, "perspective too extreme, please rescan")`

`test_reject_too_dim(tmp_path)`:
- Create a canvas `np.full((TARGET_H, TARGET_W, 3), 15, uint8)` (very dark — median ~15)
- Add 4 markers (they are black on a dark background — this is intentionally extreme for testing the dim rejection path)
- Save, call `rectify_scan`, assert result is `(False, "too dim, try again")`
- NOTE: If marker detection fails first (markers hard to see on dark bg), that is acceptable — either rejection path fires and the scan is rejected without crash

`test_reject_overexposed(tmp_path)`:
- Create a canvas `np.full((TARGET_H, TARGET_W, 3), 240, uint8)` (very bright — median ~240)
- Add 4 markers (white markers are invisible on a near-white bg — that is fine; the overexposed check may fire before or after marker detection)
- Save, call `rectify_scan`, assert result is `(False, ...)` — either "couldn't read corners, try again" or "too overexposed, try again" — EITHER is acceptable since both result in no output file and a user-facing retry prompt. Assert output file does NOT exist.
- IMPORTANT: Do NOT assert on the exact message for overexposed — the test fixture may fail marker detection before reaching histogram check. The key invariant is: no crash, no output file.

`test_no_output_on_rejection(tmp_path)`:
- For each rejection case (no markers, skew, dim), assert the output file is absent after the call. (This consolidates the file-absence checks as explicit contract.)
