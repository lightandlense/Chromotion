# Car Kiosk Process

## Required assets per car

1. **Lineart PNG** — coloring-page style, flat side profile, white fill, black outlines
   - Car should face LEFT (front on left, rear on right)
   - No wheels version: `cadillac_no_wheels.png` — transparent wheel wells + background
   - Full stencil: `cadillac_full_stencil.png` — opaque body + windows, transparent background only

2. **wheel.png** — hub/spoke graphic asset (RGBA, ~164px square)
   - Shared across cars unless a car needs a unique hub design

3. **Printable template** — lineart with ArUco corner markers for scan rectification
   - Generated by `src/offline/make_printable_template.py`

---

## Wheel geometry calibration

Find wheel center (cx, cy) and radius (r) in the 1200x896 stencil space.

Option A — Hough circle detection (automated):
```bash
python -c "
import cv2, numpy as np
img = cv2.imread('src/animations/car/cadillac_no_wheels.png', cv2.IMREAD_GRAYSCALE)
circles = cv2.HoughCircles(img, cv2.HOUGH_GRADIENT, 1, 100, param1=50, param2=30, minRadius=60, maxRadius=120)
print(np.round(circles[0]))
"
```

Option B — Visual tuner: open `src/runtime/wheel_tuner.html` and use arrow keys.

Store in `data/vehicles/<car_name>_parts.json`:
```json
{
  "WHEEL_FRONT": [234, 530, 88],
  "WHEEL_REAR":  [889, 529, 87]
}
```

## mask_car_scan.py — key decisions per car

### Wheel punch
- **Lower arc only** (`ys >= cy - r//3`): use when the car has a visible fender arch
  covering the top of the wheel well. Preserves the fender as opaque so it covers
  the top of the spinning wheel sprite.
- **Full circle** (`in_circle` only): use when the wheel sits in an open well with
  no fender arch, or when the wheel graphic needs to show fully.

### Color filter (always use these values unless crayon colors are very pale)
```python
is_colored = (s > 0.15) & (v > 0.40)   # S > 15%, V > 40% brightness
```

### Bottom-to-top mirror
Always mirror bottom half scan colors to fill top half gaps. Already in the script.

## Hub offset calibration (browser)

After generating `front_wheel_colors.png` / `rear_wheel_colors.png`:

1. Open `src/runtime/car2_kiosk_test.html`
2. F = front wheel, B = rear wheel
3. Arrow keys nudge the color layer (Shift = 10px steps)
4. When aligned: read ox/oy from debug bar
5. Bake into HTML `hubCfg` defaults and call `applyHubCfg()` on init

**Cadillac 1962 calibrated values:** front=(16,-2), rear=(-12,0)

---

## Adding a new car

1. Create `vehicles/cars/<car_name>/` with: `lineart.png`, `no_wheels.png`, `stencil.png`, `wheel.png`
2. Copy `src/offline/mask_car2_scan.py` → `vehicles/cars/<car_name>/mask_scan.py`, update constants
3. Copy `src/runtime/car2_kiosk_test.html` → `vehicles/cars/<car_name>/kiosk.html`, update paths + hubCfg
4. Calibrate wheel geometry, set punch mode, calibrate hub offsets
5. Generate static test scan, confirm colors land correctly
